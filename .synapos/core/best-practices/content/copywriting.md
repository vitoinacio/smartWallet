# Copywriting

> Guia para escrever textos que geram ação — com clareza, relevância e voz consistente.

---

## Princípios

- **Escreva para uma pessoa, não para um público** — quanto mais específico o leitor imaginado, mais eficaz o texto
- **Benefício antes de feature** — o leitor quer saber o que ganha, não como funciona
- **Clareza > criatividade** — um texto simples que comunica bem supera um texto sofisticado que confunde
- **Cada palavra tem um custo** — se não acrescenta, corta
- **Voz consistente** — o texto deve soar como a marca, sempre

---

## Estrutura AIDA

Framework clássico para estruturar copy persuasivo:

| Etapa | Objetivo | Pergunta a responder |
|-------|----------|----------------------|
| **Atenção** | Parar o scroll | "Por que eu deveria ler isso?" |
| **Interesse** | Gerar curiosidade | "Isso é relevante para mim?" |
| **Desejo** | Criar querer | "Quero isso na minha vida?" |
| **Ação** | Converter | "O que eu faço agora?" |

### Exemplo

```
[Atenção]
Você perde 2 horas por dia respondendo as mesmas perguntas.

[Interesse]
Times de suporte com mais de 3 pessoas gastam em média 40% do tempo
em perguntas repetitivas — perguntas que já foram respondidas antes.

[Desejo]
Com o Synapse, sua base de conhecimento responde por você.
Automaticamente. Sem treinamento, sem setup complexo.

[Ação]
Teste grátis por 14 dias — sem cartão de crédito.
```

## Estrutura PAS

Para copy de problema/dor — especialmente eficaz em landing pages e emails:

- **P — Problema:** Identifique a dor com precisão
- **A — Agitação:** Amplifique as consequências do problema não resolvido
- **S — Solução:** Apresente a saída de forma clara e crível

### Exemplo

```
[Problema]
Documentação desatualizada causa retrabalho e decisões erradas.

[Agitação]
Quando os docs não acompanham o produto, novos devs perdem dias
tentando entender como o sistema realmente funciona. Bugs surgem
de suposições erradas. Onboarding vira um pesadelo.

[Solução]
O Synapse mantém sua documentação viva — atualizada a cada sprint,
vinculada ao código, acessível a qualquer membro do time.
```

## Headlines

A headline decide se o resto será lido. Invista tempo aqui.

### Fórmulas que funcionam

| Fórmula | Exemplo |
|---------|---------|
| Número + benefício | "5 maneiras de reduzir o churn sem aumentar o suporte" |
| Como + resultado | "Como documentar uma API em 30 minutos" |
| Pergunta direta | "Seu time ainda usa planilha para roadmap?" |
| Promessa específica | "Reduza o tempo de onboarding em 50% no primeiro mês" |
| Antes/depois | "De docs bagunçados para base de conhecimento estruturada" |

### Checklist de headline

- [ ] Comunica o benefício principal?
- [ ] É específica (números, resultados concretos)?
- [ ] Faz o leitor querer continuar?
- [ ] Está alinhada com o que o restante do texto entrega?
- [ ] Evita jargão que o leitor pode não conhecer?

## CTA — Call to Action

O CTA é a instrução de ação. Deve ser claro, urgente e de baixo atrito.

### Boas práticas

- Use verbos no imperativo: "Comece", "Baixe", "Descubra", "Teste"
- Específico > genérico: "Ver planos" > "Saiba mais"
- Reduza o atrito aparente: "Teste grátis" > "Compre agora"
- Um CTA principal por peça — não divida a atenção

### Exemplos por contexto

| Contexto | CTA fraco | CTA forte |
|----------|-----------|-----------|
| Trial | "Clique aqui" | "Começar teste gratuito" |
| Download | "Baixar" | "Baixar o guia em PDF" |
| Contato | "Entre em contato" | "Falar com especialista" |
| Demo | "Ver mais" | "Agendar demonstração" |

## Tom de Voz

### Dimensões

| Dimensão | Definição |
|----------|-----------|
| **Formal ↔ Informal** | Trata o leitor como "você" ou "sr./sra."? |
| **Técnico ↔ Acessível** | Usa jargão do setor ou explica tudo? |
| **Direto ↔ Narrativo** | Vai direto ao ponto ou conta uma história? |
| **Sério ↔ Descontraído** | Tom corporativo ou mais próximo e humano? |

### Como manter consistência

1. Defina as 3 palavras que descrevem a voz da marca
2. Para cada palavra, liste 2 exemplos de como soa e 2 de como NÃO soa
3. Revise todo copy contra esse padrão antes de publicar

---

## Revisão e Edição

### Processo recomendado

1. **Escreva sem filtro** — dump de ideias, sem autocensura
2. **Estruture** — organize seguindo AIDA, PAS ou estrutura definida
3. **Corte pela metade** — elimine o que não acrescenta
4. **Leia em voz alta** — identifica frases que travam
5. **Revise para o leitor** — troque "nós" por "você" onde possível
6. **Revisão final** — ortografia, gramática, links

### Checklist de qualidade

- [ ] O benefício principal está claro nos primeiros 2 parágrafos?
- [ ] Há um CTA claro e único?
- [ ] O texto está na voz definida para a marca?
- [ ] Está escrito para o leitor certo?
- [ ] Sem jargão não explicado?
- [ ] Revisado por pelo menos uma segunda pessoa?
